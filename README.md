# PCB 모듈 브라켓 설계와 3D 프린팅 가이드

이 글은 제가 출력 세트를 설계하고 3D 프린터로 출력하면서 남긴 기록입니다. 스테레오 이벤트 카메라 모듈과 FPGA 보드를 고정하는 브라켓을 만들며, 처음에는 무엇을 재야 하는지, 어디를 비워야 하는지, 출력 전에 무엇을 확인해야 하는지 하나씩 정리했습니다.

처음 3D 프린팅으로 PCB 브라켓을 만들 때 바로 쓸 수 있도록, 제가 실제로 사용한 치수 기준·체결 방식·슬라이서 설정과 출력 뒤에 확인한 내용을 함께 적었습니다. 특정 보드에만 맞는 치수도 포함되어 있으므로, 다른 장비에 적용할 때는 그 값을 그대로 복사하지 말고 실제 부품에서 다시 확인해야 합니다.

## 1. 준비물과 기록할 것

### 준비물

- 고정하려는 실제 PCB와 연결 케이블
- PCB에 맞는 나사와 너트
- 캘리퍼스 또는 정밀 자
- 3D 프린터와 필라멘트
- 작은 나사 구멍 시험편을 출력할 재료

회사에 보관된 나사 묶음에서는 실제 이벤트 카메라 모듈과 FPGA 보드를 들고 가서 비교했습니다. 나사산, 머리 형상, 실제 길이는 사진이나 규격명만으로 오차가 생기기 쉽습니다. 맞는 나사를 찾으면 직경과 필요한 체결 길이를 함께 기록해 두는 편이 좋습니다.

### CAD에 넣기 전에 측정할 치수

보드의 가로·세로만 재면 충분하지 않습니다. 아래 값이 브라켓 설계의 기준이 됩니다.

| 확인할 항목 | 왜 필요한가 |
| --- | --- |
| PCB 외곽 크기와 두께 | 레일 폭, 끼움 높이, 보드 외곽 간섭을 정합니다. |
| 나사 구멍의 중심 좌표와 직경 | 고정 보스와 체결 구멍의 정확한 기준입니다. 원의 바깥쪽 거리만 재면 안 되고 중심 간격으로 바꿔야 합니다. |
| 렌즈 중심 또는 기준축 위치 | 스테레오 baseline, 촬영 방향, 다른 부품과의 높이 관계를 맞춥니다. |
| 커넥터·USB·FFC의 위치와 높이 | 브라켓이 포트와 케이블을 막지 않도록 합니다. |
| 케이블의 실제 굽힘 방향과 필요한 공간 | 커넥터 바로 앞뿐 아니라 케이블이 빠져나가는 방향 전체를 비워야 합니다. |
| 나사 머리·너트의 크기와 체결 방향 | 뒤쪽 너트 공간, 드라이버 접근, 자리파기 필요 여부를 판단합니다. |
| 프린터 공차 | CAD 구멍과 실제 출력 구멍은 다르므로 체결 시험값이 필요합니다. |

나사 구멍 간격은 가능한 한 중심 간격을 직접 측정했습니다. 외곽 간격만 측정했다면 `중심 간격 = 외곽 간격 + 구멍 지름`으로 환산합니다.

## 2. 사용하는 프로그램과 역할

### FreeCAD

FreeCAD는 mm 단위의 정확한 3D 형상, 구멍, 레일, 리브를 만들고 수정할 때 사용했습니다. PCB 브라켓은 구멍 위치나 보드 두께가 조금만 바뀌어도 형상을 다시 맞춰야 하므로, 치수를 파라미터로 관리하는 방식이 편합니다.

작업은 다음 순서로 진행했습니다.

1. 실제 보드의 단순 더미 모델을 만듭니다.
2. 나사 구멍 중심, 커넥터, 케이블 공간을 기준점으로 둡니다.
3. 더미 모델을 기준으로 고정대 형상을 만듭니다.
4. 조립 상태에서 간섭과 체결 방향을 확인합니다.
5. 필요한 부품만 STL로 내보냅니다.

FreeCAD Python은 반복되는 보스, 구멍 grid, 레일, 여러 baseline 위치처럼 규칙적인 형상을 만들 때 사용했습니다. 치수 값을 스크립트 상단에 모아 두면 다음 프로젝트에서 보드만 바꾸어 다시 쓰기 쉽습니다.

### 3DWOX Desktop / 3DWOX Slicer

3DWOX Desktop은 STL을 3DWOX 2X가 읽는 G-code로 바꾸는 슬라이서입니다. 출력 방향, 레이어 두께, 서포트, 필라멘트 온도, 인필을 설정하고 레이어 미리보기로 실제 출력 경로를 확인했습니다.

STL은 형상 파일일 뿐 프린터 동작 조건이 들어 있지 않습니다. 프린터에 넣을 파일은 같은 프린터와 재료 프로파일로 슬라이스한 G-code를 쓰는 것이 안전합니다. G-code는 프린터와 프로파일에 따라 달라지므로 다른 제조사 프린터에 그대로 쓰지 않습니다.

### 3D 뷰와 렌더 확인

FreeCAD 조립 뷰 또는 STL의 정면·측면·상면·축측 뷰를 확인했습니다. 특히 아래 항목은 정면만 보면 놓치기 쉽습니다.

- 나사 구멍이 보스나 리브에 막혔는지
- 너트를 끼울 뒤쪽 공간이 있는지
- 보드가 레일의 홈 바닥면에 실제로 닿는지
- 케이블 출구가 브라켓과 겹치지 않는지
- 길게 뻗은 지지대가 공중에 떠 있지 않은지
- 출력 방향에서 지붕 아래나 보스 아래에 서포트가 필요한지

## 3. 실제 부품을 단순 3D 모델로 만드는 방법

처음부터 부품의 실물 외관을 전부 모델링할 필요는 없습니다. 고정 구조에 영향을 주는 요소만 정확하면 충분합니다.

- PCB: 외곽 사각형과 실제 두께
- 나사 구멍: 중심 위치와 관통 구멍
- 렌즈: 중심축을 확인할 수 있는 원통
- 커넥터: 가장 튀어나온 높이와 폭을 가진 단순 박스
- USB·전원 포트: 플러그가 들어가는 방향을 포함한 단순 박스
- FFC 케이블: 커넥터에서 시작해 실제로 꺾이는 방향을 포함한 여유 공간 형상

케이블은 선 한 가닥으로만 생각하면 안 됩니다. FFC는 폭과 굽힘 반지름을 가진 띠로, USB와 전원 케이블은 커넥터보다 큰 플러그와 굽힘 공간을 가진 부피로 잡았습니다. 이를 `keepout` 형상으로 두면 브라켓을 만들 때 충돌 여부를 눈으로 확인할 수 있습니다.

스테레오 카메라처럼 기준축이 있는 부품은 렌즈 중심을 별도 기준점으로 둡니다. 보드 외곽이 아니라 렌즈 중심 사이 거리가 baseline이므로, 카메라가 옮겨져도 그 기준으로 거리와 수평을 확인할 수 있습니다.

## 4. 브라켓 형상 설계 원칙

### 접촉하는 곳만 잡기

PCB 전체를 큰 판으로 덮으면 재료와 시간이 많이 들고, 포트와 케이블도 막기 쉽습니다. 이번에는 아래 조합으로 만들었습니다.

- PCB 아래를 받는 하단 레일 또는 홈
- 나사 구멍을 잡는 보스 2개 또는 4개
- 긴 구조를 지지하는 얇은 리브
- 필요한 위치의 너트 공간과 드라이버 접근 공간

카메라 홀더는 하단 레일과 두 개의 나사 보스로 보드를 기준 위치에 잡고, 뒤쪽은 필요한 리브만 두는 모듈형 구조로 만들었습니다. 보드마다 홀더만 교체하고 베이스는 재사용할 수 있다는 점이 장점입니다.

### 레일과 기준면

카메라 PCB의 바닥면은 레일 외형의 가장 아래 바닥이 아니라, **하단 레일 홈에서 PCB가 실제로 닿는 홈 바닥면**에 닿아야 합니다. 이 홈 바닥면을 높이 기준면으로 정하고, 카메라 나사 구멍과 렌즈 중심 높이를 모두 이 기준면에서 계산했습니다.

보드가 홈 위에 떠 있거나 홈이 보드 아래로 과도하게 내려가면, 나사 높이가 맞아도 실제 조립 자세가 틀어집니다. 이번 설계에서는 하단 레일 홈 깊이 `5.00 mm`, 홈 바닥에서 카메라 나사 중심까지 `38.985 mm`라는 관계를 사용했습니다. 이 수치는 다른 PCB에 그대로 적용하는 규격이 아니라, 레일 기준면에서 나사 위치를 계산하는 예시입니다.

레일은 너무 깊으면 보드를 끼우고 빼기 어렵고, 너무 얕으면 흔들립니다. 이번 카메라 홀더는 하단 레일 홈 깊이 `5.00 mm`를 사용했습니다. 다른 PCB에서는 보드 두께, 삽입 방향, 케이블 방향을 보고 다시 정해야 합니다.

### 나사 보스와 고깔형 접촉부

PCB 나사 구멍을 잡는 접촉부는 단순 원통보다 앞은 얇고 뒤는 넓어지는 고깔형 보스로 만들었습니다.

- PCB에 닿는 앞쪽: 나사 구멍 주변만 얇게 잡아 부품과 간섭을 줄입니다.
- 뒤쪽: 원통 또는 테이퍼를 넓혀 힘을 분산합니다.
- 보스 중심: 나사가 완전히 관통할 수 있어야 합니다.
- 뒤쪽: 너트를 넣을 공간과 드라이버가 접근할 공간을 비웁니다.

보스의 바깥 두께는 앞쪽에서 얇아도 되지만, 길게 튀어나온 부분 전체를 얇게 만들면 쉽게 휩니다. 앞쪽 접촉 ring은 필요한 최소 두께로 두고, 보드 뒤쪽으로 갈수록 넓혀 리브나 지지판에 연결했습니다. 나사 구멍 뒤쪽에는 상관없는 기둥이 관통하지 않도록 했습니다.

### 리브와 두께

출력물은 생각보다 너무 두껍게 만들 필요가 없었습니다. `0.4 mm` 노즐 기준으로 벽 두께를 최소 `1.2 mm` 이상, 하중이 걸리는 리브·보스 주변은 `1.6~2.4 mm` 정도로 시작하면 PLA에서도 충분히 단단한 경우가 많았습니다. 큰 판을 통째로 두껍게 하는 대신 다음처럼 보강했습니다.

- 길게 떠 있는 판 아래에 삼각 리브를 추가합니다.
- 보스 뒤쪽을 리브나 짧은 수평 연결대로 받칩니다.
- 양끝 지지대는 바닥과 수평 또는 사선으로 연결합니다.
- 기둥 사이에는 필요하면 얇은 가로 연결대를 둡니다.

리브는 나사 구멍과 너트, 케이블 통로를 막지 않는 위치에 둬야 합니다. 디자인이 단순해 보여도, 조립 방향과 공구 접근 공간까지 같이 보면서 결정했습니다.

### 모듈화

브라켓은 하나의 큰 출력물보다 기능별로 나누는 편이 좋았습니다.

- 카메라 홀더: 카메라 한 개의 레일과 나사 보스만 담당
- FPGA 보드 베이스: 보드 고정과 카메라 홀더 위치 조정 담당
- 결합부: 같은 피치의 구멍 grid와 공용 나사로 연결

이렇게 나누면 카메라 홀더만 다시 출력할 수 있고, 다른 카메라를 쓰게 되었을 때 베이스는 유지한 채 홀더만 새로 설계할 수 있습니다. 출력 실패도 작은 부품에서 먼저 잡을 수 있어 재료 낭비가 줄어듭니다.

## 5. 이번 모듈에서 확인한 체결 기준

아래 값은 이번 이벤트 카메라 모듈과 FPGA 보드에 사용한 CAD 시작값입니다. 프린터 상태, 나사 종류, 필라멘트 수축에 따라 달라질 수 있으므로 다른 모듈에 그대로 복사하기보다 작은 시험편으로 먼저 확인해야 합니다.

| 용도 | CAD 구멍 또는 구조 기준 | 사용 시 주의 |
| --- | --- | --- |
| 카메라 PCB 고정 나사 | 관통 구멍 `3.15 mm` | 실제 나사와 너트를 가져와 시험편에서 먼저 체결합니다. PCB 접촉부는 고깔형 보스로 만듭니다. |
| FPGA PCB 고정 나사 | 관통 구멍 `2.45 mm` | 너무 헐겁지 않게 잡은 값입니다. 보드의 네 귀퉁이와 포트·FFC 간섭을 함께 확인합니다. |
| 홀더와 베이스 결합 | `5.00 mm` 계열 구멍 | 앞뒤 두 점 이상으로 잡고, 홀더가 회전하지 않도록 간격을 둡니다. |
| 삼각대 또는 바닥 고정 | `5.00 mm` 관통 구멍 | 나사 머리나 삼각대 플레이트 형상이 필요할 때만 자리파기를 추가합니다. |
| 카메라 하단 레일 | 깊이 `5.00 mm` | 홈 바닥을 카메라 PCB 바닥 기준으로 사용합니다. |

카메라 고정 나사의 중심 간격은 이번 모듈에서 `26.61 mm`로 사용했습니다. 이 값은 해당 카메라 모듈용이므로, 다른 보드는 실제 나사 중심 간격으로 교체해야 합니다. FPGA의 네 고정 위치, USB 포트, MIPI/FFC 커넥터, 후면 포트 쪽 케이블 배출 공간은 조립 더미 모델에서 매번 확인했습니다.

## 6. 나사와 레일은 작은 시험편부터 출력하기

전체 브라켓을 출력하기 전에 아래 요소만 포함한 작은 시험편을 만들었습니다.

- 목표 구멍 지름을 `0.1~0.2 mm`씩 바꾼 나사 구멍 3~5개
- 실제 나사와 너트가 들어가는 보스 한 개
- PCB 두께에 맞춘 짧은 레일 조각
- 필요하면 케이블이 통과하는 홈 한 개

시험편으로 아래를 확인했습니다.

1. 나사가 손으로 들어가는지, 너무 헐겁지 않은지
2. 너트가 뒤쪽에서 들어가고 도구로 조일 수 있는지
3. PCB가 레일 홈에 얹혔을 때 흔들리지 않는지
4. 레일이 보드 모서리를 과도하게 누르지 않는지
5. FFC 또는 USB 플러그가 실제 방향으로 빠져나오는지

이 단계를 건너뛰면 큰 브라켓에서 구멍 하나가 조금 틀어진 이유로 전체를 다시 출력하게 됩니다. 체결 시험편은 짧게는 수십 분 안에 결과를 볼 수 있어 가장 효율적인 확인 방법이었습니다.

## 7. 간섭과 조립 조건 점검표

STL을 내보내기 전에 아래를 모두 확인합니다.

- [ ] PCB 나사 구멍 중심과 브라켓 보스 중심이 일치하는가
- [ ] 나사가 보스를 완전히 관통하는가
- [ ] 나사 뒤쪽에 너트·와셔·드라이버 공간이 있는가
- [ ] 보드 바닥이 **하단 레일 홈 바닥면**에 실제로 닿는가
- [ ] 레일이 커넥터, 납땜부, 렌즈 하우징을 누르지 않는가
- [ ] FFC가 빠져나갈 위쪽·옆쪽 공간이 충분한가
- [ ] USB, 전원, MIPI/FFC 포트의 플러그 삽입 방향이 열려 있는가
- [ ] 두 카메라의 렌즈 중심이 같은 높이와 방향인가
- [ ] baseline은 렌즈 중심 기준으로 계산했는가
- [ ] 리브와 보스가 나사 구멍·너트·케이블 통로를 막지 않는가
- [ ] 출력 방향에서 공중에 뜨는 지붕, 보스, 레일 아래에 서포트가 생성되는가

조립 상태를 렌더한 뒤 정면, 측면, 위, 뒤 네 방향에서 위 목록을 확인하는 습관이 필요합니다. 한 방향에서만 정상으로 보여도 반대쪽에서 너트 공간이 막힌 경우가 자주 있었습니다.

## 8. Sindoh 3DWOX 2X 출력 가이드

### 프린터 사양과 크기 제한

사용한 프린터는 Sindoh 3DWOX 2X입니다. FFF 방식, `0.4 mm` 노즐, `1.75 mm` 필라멘트를 사용합니다.

| 항목 | 기준 |
| --- | --- |
| 최대 출력 크기 | 노즐 1: `228 × 200 × 300 mm`, 노즐 2: `225 × 200 × 300 mm` |
| 노즐 직경 | `0.4 mm` |
| 필라멘트 직경 | `1.75 mm` |
| 레이어 높이 범위 | `0.05~0.40 mm` |
| 일반 브라켓 시작 레이어 | `0.20 mm` |
| 일반 출력 속도 | `40 mm/s` |

긴 브라켓은 평평하게 놓으면 베드의 `228 mm` 방향을 넘을 수 있습니다. 이 경우 가장 긴 축을 Z 방향으로 세우고, Z 높이 `300 mm` 안에 들어가는지 먼저 확인합니다. 세워서 출력하면 서포트와 raft가 더 중요해집니다.

### PLA 기본 프로파일

아래 값은 이번 PLA 브라켓 출력에서 사용한 3DWOX Desktop 프로파일과 생성된 G-code를 기준으로 정리했습니다. 다른 재료를 쓸 때는 해당 재료 프로파일을 우선합니다.

| 설정 | PLA 브라켓 시작값 |
| --- | --- |
| 노즐 온도 | `200 C` |
| 베드 온도 | `60 C` |
| 레이어 높이 | `0.20 mm` |
| 초기 레이어 높이 | `0.20 mm` |
| 벽 두께 | `0.80 mm 이상` |
| 상·하단 두께 | `0.80 mm 이상` |
| 인필 | `15%`, 첫 실사용 브라켓은 `20%`부터 확인 |
| 일반 출력 속도 | `40 mm/s` |
| 초기 레이어 속도 | `20 mm/s` |
| 이동 속도 | `130 mm/s` |
| 리트랙션 | 사용, `6.00 mm / 30 mm/s` |
| 냉각 팬 | 사용, `50%` |
| 최소 레이어 시간 | `5초` |

### 서포트와 베드 부착 설정

브라켓처럼 레일, 지붕, 고깔 보스가 있는 형상은 서포트를 켭니다. 처음에는 다음 설정으로 시작하고, 슬라이서 미리보기에서 필요한 부분만 조정합니다.

| 설정 | 시작값 |
| --- | --- |
| 서포트 | 켬 |
| 배치 | 우선 `Touching buildplate`, 내부 지붕이 뜨면 `Everywhere` |
| 패턴 | `Zigzag` |
| overhang 각도 | `60도` |
| 서포트 밀도 | `20%` |
| XY 간격 | `0.80 mm` |
| Z 간격 | `0.20 mm` |
| 베드 부착 | `Raft` |
| Raft 여유 | `5.00 mm` |
| Raft base / interface / surface 속도 | `20 mm/s` |

`Everywhere` 서포트는 내부까지 잘 받쳐 주지만, 작은 구멍이나 케이블 홈에 서포트가 남을 수 있습니다. 레이어 미리보기에서 FFC 창, 나사 구멍, 너트 공간이 모델 재료나 서포트로 막히지 않았는지 꼭 확인합니다.

### 슬라이싱과 출력 절차

1. 3DWOX Desktop에서 STL을 엽니다.
2. 프린터를 `3DWOX 2X`, 재료를 PLA로 선택합니다.
3. 모델이 베드 크기와 높이 제한 안에 들어가는지 확인합니다.
4. 출력 방향을 정하고 서포트와 raft를 적용합니다.
5. 슬라이스 후 레이어 미리보기로 바닥, 레일, 보스, 구멍, 지붕을 확인합니다.
6. G-code로 저장합니다.
7. 같은 Wi-Fi에 연결된 경우 앱에서 프린터를 탐지해 전송하거나, USB에 G-code를 넣어 프린터에서 선택합니다.

USB 출력도 가능하지만, STL을 그대로 넣는 방식보다 3DWOX 2X와 해당 PLA 프로파일로 만든 G-code를 넣는 방식으로 출력했습니다. 다른 프린터용 G-code는 시작 명령, 온도 제어, 베드 크기, 펌웨어 명령이 달라 호환되지 않을 수 있습니다.

### 장기간 미사용 후 점검

프린터를 1~2주 쓰지 않았다면 필라멘트가 내부에서 끊어졌거나 노즐 공급이 불안정할 수 있습니다. 큰 출력 전에 아래를 확인합니다.

- 필라멘트가 정상적으로 로딩되어 있는지
- 노즐에서 재료가 일정하게 나오는지
- 이전 재료가 남아 있지 않은지
- 베드가 깨끗하고 raft가 붙을 상태인지

처음 몇 cm의 토출이 불안하면 바로 큰 출력을 시작하지 말고 purge나 작은 시험 출력부터 합니다.

## 9. 재료와 시간을 줄이는 방법

- PCB 전체를 덮는 큰 판 대신 레일, 보스, 리브를 사용합니다.
- 하중이 적은 넓은 면에는 큰 관통 창을 두되, 나사·리브·케이블 주변은 남깁니다.
- 고정대는 베이스와 홀더로 분리해 변경된 부품만 다시 출력합니다.
- 첫 출력은 인필 `15~20%`와 필요한 벽 두께로 시작합니다. 강도가 부족한 지점만 리브나 보스 두께를 늘립니다.
- 얇은 벽은 노즐 폭의 정수배에 가깝게 정합니다. `0.4 mm` 노즐에서는 `1.2 mm`, `1.6 mm`, `2.0 mm` 같은 두께가 예측하기 쉽습니다.
- 긴 다리나 보스는 두껍게 만들기보다 삼각 리브로 받치면 재료를 덜 씁니다.

## 10. 작업 순서

1. 실제 PCB, 나사, 케이블을 준비하고 필요한 치수를 측정합니다.
2. PCB·구멍·커넥터·케이블 공간만 담은 더미 모델을 만듭니다.
3. 레일 홈 바닥과 렌즈 중심 같은 조립 기준면을 먼저 정합니다.
4. 나사 보스와 레일만 들어간 작은 시험편을 출력합니다.
5. 실제 나사·너트·PCB로 체결과 간섭을 확인합니다.
6. 카메라 홀더와 FPGA 베이스처럼 기능 단위로 모델을 나눕니다.
7. 조립 3D 뷰에서 나사 관통, 너트 공간, 포트, FFC, baseline을 확인합니다.
8. STL을 내보내고 3DWOX Desktop에서 방향·서포트·raft를 설정합니다.
9. 레이어 미리보기에서 모든 구멍과 케이블 창이 열린 상태인지 확인합니다.
10. 작은 부품부터 출력하고, 맞으면 전체 조립품을 출력합니다.

## 마무리

PCB 브라켓은 치수만 맞춘 상자가 아니라 실제 조립 동작을 담는 부품입니다. 보드가 어디에 닿는지, 나사를 어느 쪽에서 넣는지, 너트와 드라이버가 어디에 들어가는지, 케이블이 어느 방향으로 빠져야 하는지를 먼저 모델에 넣어야 합니다.

이번 스테레오 카메라 거치대에서는 실제 부품의 단순 더미 모델을 먼저 만들고, 나사 보스와 하단 레일을 시험 출력한 뒤, 카메라 홀더와 FPGA 베이스를 분리한 방식이 가장 효과적이었습니다. 이 흐름을 지키면 다음 프로젝트에서도 실패한 전체 출력 횟수와 재료 낭비를 줄일 수 있습니다.

아래 출력 세트는 FPGA 베이스 1개와 카메라 홀더 2개를 한 세트로 구성한 예시입니다. 조립할 때는 FPGA 보드를 베이스 후면의 네 고정점에 나사로 체결하고, 카메라 홀더 두 개는 베이스의 반복 결합 구멍 중 원하는 위치에 각각 고정합니다. 홀더 위치를 바꾸면 두 카메라 렌즈 사이의 baseline을 바꿀 수 있으며, 보드가 바뀌면 카메라 홀더만 새로 출력할 수 있습니다.

### 실제 고정 대상과 케이블 출구

실제 조립에서는 카메라 렌즈가 정면을 향하고, 카메라 고정 나사 구멍이 렌즈 위쪽에 오도록 둡니다. FFC 케이블은 카메라 상단 방향으로 빠져나갈 공간을 확보해야 하며, FPGA의 USB·FFC·전원 포트 쪽도 브라켓이나 케이블이 막지 않도록 비워 둡니다.

### 카메라 홀더가 보드를 잡는 방식

카메라 PCB는 아래쪽 중앙의 레일 홈에 먼저 얹힙니다. 홈 바닥이 보드의 기준면이므로, 카메라가 아래로 처지거나 나사만으로 떠 있는 구조가 아닙니다. 레일은 보드 전체를 감싸지 않고 중앙 구간만 잡아 커넥터, 케이블, 양옆 부품을 피합니다.

상단의 두 고깔형 보스는 PCB 나사 구멍 주변에만 닿아 고정합니다. 보스 중심 구멍은 나사가 완전히 관통하도록 만들고, 뒤쪽은 너트가 들어갈 수 있도록 비웁니다. 고깔의 앞쪽은 간섭을 줄이기 위해 얇고, 뒤쪽은 리브와 지지대로 연결해 출력 강도를 확보했습니다. 즉, 보드와 닿는 부분은 작게, 힘을 받는 부분은 뒤에서 넓게 받는 구조입니다.

### FPGA 베이스가 하는 일

FPGA 베이스는 보드의 네 나사 위치를 기준으로 FPGA를 지지하고, 앞쪽의 반복 결합 구멍 grid로 두 카메라 홀더 위치를 조절합니다. 카메라 홀더는 앞뒤 두 줄의 구멍을 사용해 네 점으로 고정하므로, 단순히 한 줄 구멍에 끼운 구조보다 회전과 흔들림에 강합니다. 중앙의 바닥 고정 구멍은 삼각대 또는 다른 장착판에 연결할 때 사용합니다.

FPGA 쪽은 넓은 판으로 보드 전체를 덮지 않고, 나사 고정점과 필요한 지지 구조만 남겼습니다. 이 방식은 포트와 케이블의 접근성을 유지하면서 출력 시간과 필라멘트 사용량을 줄입니다.

### 한 번에 출력하는 배치

한 번에 출력할 때는 FPGA 베이스 1개와 같은 카메라 홀더 2개를 한 STL에 배치할 수 있습니다. 베이스의 긴 방향은 프린터 높이 방향으로 세워 베드 제한 안에 넣고, 카메라 홀더는 베이스 옆의 빈 공간에 배치합니다. 이 방향은 베드 폭을 아끼는 대신 레일·보스·지붕 아래에 서포트가 필요하므로, 앞의 PLA 서포트와 raft 설정을 적용한 뒤 레이어 미리보기로 확인합니다.

사진이나 결과물을 문서에 추가할 때는 아래 순서가 이해하기 좋았습니다.

1. 실제 보드와 케이블이 놓인 앞면 또는 뒷면 사진
2. 카메라 홀더 단품 사진 또는 CAD 렌더
3. FPGA 베이스 단품 사진 또는 CAD 렌더
4. 출력 세트 배치 사진
5. 실제 조립 후 정면·후면 사진

각 사진에는 단순히 부품 이름만 쓰기보다, `레일이 보드 아래를 받는다`, `고깔 보스는 나사 구멍 주변만 지지한다`, `이 공간은 FFC 케이블 출구다`, `이 구멍은 홀더 위치 조정용이다`처럼 설계 의도를 한 줄로 같이 적는 편이 좋았습니다.
