# 스테레오 이벤트 카메라 · FPGA 브라켓 3D 프린팅 기록

스테레오 이벤트 카메라 모듈 2개와 FPGA 보드를 한 구조에 고정하기 위해 만든 모듈형 브라켓의 설계·출력 기록입니다. 이 문서는 **V4 출력 세트**를 기준으로, 실제 부품을 어떻게 재고, CAD에서 어떤 기준을 잡았고, Sindoh 3DWOX 2X로 어떤 순서로 출력했는지를 정리합니다.

처음부터 큰 브라켓을 한 번에 출력하기보다, 실제 PCB와 나사·케이블을 기준으로 더미 모델을 만들고 작은 체결 시험편을 먼저 출력했습니다. 이 순서가 맞아야 전체 출력에서 구멍, 레일, 케이블 공간 때문에 다시 만드는 일을 줄일 수 있었습니다.

> 이 저장소의 STL과 사진은 특정 이벤트 카메라 모듈 및 FPGA 보드 조합을 기준으로 합니다. 다른 모듈을 사용할 때는 치수와 체결 조건을 그대로 복사하지 않고, 실제 보드·나사·케이블로 다시 확인해야 합니다.

## 출력물 개요

</p>

출력 세트는 세 부분으로 나누었습니다.

1. **카메라 홀더 2개**: 이벤트 카메라 모듈의 PCB를 하단 레일 홈으로 받치고, 나사 보스로 위치를 잡습니다.
2. **FPGA 보드 베이스 1개**: FPGA 보드의 네 고정점을 지지하고, 앞쪽 결합 구멍 grid에 두 카메라 홀더를 연결합니다.
3. **결합부와 바닥 고정부**: 카메라 간 렌즈 중심 거리 조절과 삼각대·장착판 결합에 사용합니다.

카메라 간 거리는 PCB 외곽이 아니라 **렌즈 중심 간 거리**로 확인했습니다. 카메라가 정면을 보도록 먼저 방향을 정한 뒤, 고정 나사가 렌즈 위쪽에 오고 FFC 케이블이 위쪽으로 빠져나오도록 홀더의 개구부를 잡았습니다.

</p>

후면에서는 FFC, USB, 전원 커넥터가 막히지 않는지가 핵심입니다. 브라켓은 PCB 전체를 덮지 않고, 지지에 필요한 레일·보스·리브만 남겼습니다.

## 시작 전에 측정한 값

PCB 고정용 브라켓은 보드의 가로·세로만 맞추면 끝나지 않습니다. 아래 값이 CAD의 기준점이 됩니다.

| 확인할 항목 | 설계에서 쓰는 이유 |
| --- | --- |
| PCB 외곽 크기와 두께 | 레일 폭, 끼움 높이, 외곽 간섭을 정합니다. |
| 나사 구멍 중심 좌표·직경 | 보스 위치와 관통 구멍의 기준입니다. |
| 렌즈 중심 또는 기준축 위치 | 스테레오 baseline과 촬영 방향을 맞춥니다. |
| USB·전원·MIPI/FFC 커넥터 위치와 높이 | 브라켓과 플러그가 겹치지 않게 합니다. |
| 케이블의 굽힘 방향과 여유 공간 | 커넥터 바로 앞뿐 아니라 케이블이 빠져나가는 경로를 비웁니다. |
| 나사 머리·너트·드라이버 공간 | 체결 방향과 자리파기 필요 여부를 판단합니다. |
| 프린터 공차 | CAD 구멍과 실제 출력 구멍의 차이를 시험편으로 보정합니다. |

나사 구멍은 외곽 거리보다 **중심 간격**이 중요합니다. 외곽 간격만 잰 경우에는 `중심 간격 = 외곽 간격 + 구멍 지름`으로 바꿔 사용했습니다.

## FreeCAD와 3DWOX Desktop의 역할

### FreeCAD: 치수와 조립 조건을 관리하는 CAD

FreeCAD에서는 mm 단위로 PCB 외곽, 두께, 나사 구멍, 레일, 보스, 리브를 만들었습니다. 실제 부품의 외관을 전부 재현하지 않고, 조립에 영향을 주는 부분만 단순 더미 모델로 만들었습니다. 그 더미와 브라켓을 조립 위치에 놓으면 간섭을 빨리 확인할 수 있습니다.

- PCB 두께와 외곽, 나사 구멍 중심을 먼저 잡습니다.
- 렌즈 중심을 별도 기준점으로 두고, baseline은 PCB 외곽이 아니라 렌즈 중심 간 거리로 확인합니다.
- USB·전원·MIPI/FFC 커넥터는 플러그 삽입 방향과 플러그 몸체까지 포함한 단순 박스로 둡니다.
- 반복되는 결합 구멍 grid, 보스, 여러 카메라 위치는 FreeCAD Python으로 만들면 치수를 상단 파라미터로 모아 다시 만들기 쉽습니다.
- STL을 내보내기 전 정면·측면·상면·후면·축측 뷰를 확인합니다. 이때 나사 구멍 막힘, 너트 공간, 레일 홈 바닥 접촉, FFC/USB 출구를 봅니다.

### 3DWOX Desktop: 출력 조건을 G-code로 만드는 슬라이서

STL에는 형상만 들어 있습니다. 프린터가 실제로 읽는 파일은 **3DWOX 2X와 해당 PLA 프로파일로 슬라이스한 G-code**입니다. 3DWOX Desktop에서 출력 방향, 레이어 높이, 벽 수, 인필, 서포트, raft, 온도, 속도를 정하고 레이어 미리보기로 실제 경로를 확인했습니다.

다른 프린터에서 만든 G-code는 그대로 쓰지 않았습니다. 시작 명령, 베드 크기, 온도 제어, 펌웨어 명령이 프린터마다 다르기 때문입니다. STL은 같은 형상을 다시 쓸 수 있지만, G-code는 사용하는 프린터와 재료 프로파일에 맞춰 다시 만들어야 합니다.

## 부품 더미 모델과 케이블 keepout

처음부터 실물 외관을 복잡하게 모델링할 필요는 없습니다. 고정 구조에 영향을 주는 요소만 정확하면 됩니다.

- **PCB**: 외곽 사각형과 실제 두께
- **나사 구멍**: 중심 위치와 관통 구멍
- **렌즈**: 중심축을 확인할 수 있는 원통
- **커넥터**: 가장 튀어나온 높이와 폭을 가진 단순 박스
- **USB·전원 포트**: 플러그가 들어가는 방향을 포함한 단순 박스
- **FFC 케이블**: 커넥터에서 시작해 실제로 꺾이는 방향까지 포함한 여유 공간

FFC는 선 한 가닥이 아니라 폭과 굽힘 반지름을 가진 띠로 봤습니다. USB와 전원 케이블도 커넥터보다 큰 플러그 몸체와 굽힘 공간을 가지므로, 모두 `keepout` 형상으로 두고 브라켓과 겹치지 않게 했습니다.

카메라 고정 나사는 실제 촬영 방향을 정한 뒤 렌즈 위쪽에 오도록 배치했습니다. 이 상태에서 FFC 출구가 열려 있는지까지 확인해야 조립 후 케이블이 브라켓에 눌리지 않습니다.

## 카메라 홀더와 FPGA 베이스 설계

### 하단 레일 홈과 높이 기준면

카메라 PCB의 바닥면은 레일 외형의 가장 아래 바닥이 아니라, **하단 레일 홈에서 PCB가 실제로 닿는 홈 바닥면**에 닿아야 합니다. 이 홈 바닥면을 높이 기준면으로 정하고, 카메라 나사 구멍과 렌즈 중심 높이를 모두 여기서 계산했습니다.

보드가 홈 위에 떠 있거나 홈이 보드 아래로 과도하게 내려가면, 나사 높이가 맞아도 실제 조립 자세가 틀어집니다. V4에서는 하단 레일 홈 깊이를 `5.00 mm`, 홈 바닥에서 카메라 나사 중심까지의 관계를 `38.985 mm`로 사용했습니다. 이 값은 다른 PCB에 그대로 쓰는 규격이 아니라, **레일 기준면에서 나사 위치를 계산하는 예시**입니다.

하단 중앙 레일은 PCB를 아래에서 받는 위치 기준 역할을 합니다. 보드 전체를 감싸지 않아서 양옆의 부품과 케이블을 피할 수 있습니다.

### 고깔형 보스와 리브

카메라 나사 위치는 고깔형 관통 보스로 고정했습니다. PCB와 닿는 앞쪽은 좁게 두어 부품 간섭을 줄이고, 뒤쪽은 넓혀 리브에 연결해 강도를 확보했습니다.

- 카메라 보스의 CAD 관통 구멍 시작값은 `3.15 mm`입니다.
- 보스 중심 간격은 이번 카메라 모듈에서 `26.61 mm` 피치로 반복됩니다.
- 보스 뒤쪽에는 너트와 드라이버가 들어갈 빈 공간을 남깁니다.
- 보스 바로 뒤로 넓은 벽을 연장하면 PCB 뒤쪽 부품·케이블과 겹칩니다. 앞쪽 접촉부와 뒤쪽 리브를 분리해 설계했습니다.

앞쪽 접촉 ring은 필요한 최소 두께로 두고, 힘을 받는 뒤쪽은 리브 또는 짧은 수평 연결대로 넓게 받았습니다. 리브는 나사 구멍, 너트, 케이블 통로를 막지 않는 위치에만 넣었습니다.

### FPGA 베이스와 모듈화

카메라 홀더와 FPGA 베이스는 분리했습니다. 카메라 종류가 바뀌면 홀더만 다시 출력하고, FPGA 베이스는 그대로 유지할 수 있기 때문입니다.

- 카메라 홀더는 베이스의 앞뒤 두 줄 결합 구멍에 **네 점**으로 고정합니다. 한 줄만 쓰는 방식보다 회전과 흔들림에 강합니다.
- V4 베이스는 `2줄 × 18개`, 총 `36개`의 `5.00 mm` 결합 구멍으로 홀더 위치를 조절합니다.
- FPGA PCB 고정 관통 구멍의 CAD 시작값은 `2.45 mm`입니다.
- FPGA 보드의 네 귀퉁이 고정점과 USB·FFC·전원 포트의 출구는 더미 모델에서 함께 확인했습니다.
- 삼각대 또는 바닥 결합용 구멍은 `5.00 mm` 관통 구멍을 사용했습니다. 자리파기는 실제 나사 머리 또는 플레이트 형상 때문에 필요할 때만 넣습니다.

| 사진 번호 | 사용 위치 | 실제 조립에서 확인한 방법 |
| --- | --- | --- |
| 1 | 가장 큰 모듈·베이스 고정 나사 | 조립된 전체 모듈에서 하나를 빼고, 머리 모양·직경·길이가 같은 나사를 찾았습니다. |
| 2 | 이벤트 카메라 모듈 고정 나사 | 카메라 PCB 구멍, 보스, 너트를 함께 시험편에서 체결했습니다. |
| 3 | FPGA 보드 고정 나사 | FPGA 고정 구멍과 포트·FFC 간섭을 같이 확인했습니다. |

회사에 보관된 나사가 묶음으로 있을 때는 규격 이름만 보고 고르지 않았습니다. **조립되어 있는 부품에서 나사 하나를 먼저 빼고, 머리 형상·직경·길이가 같은 나사를 찾아 사용했습니다.** 나사산이 맞아도 길이가 너무 길면 PCB 뒤쪽 부품, 케이블, 너트 공간과 겹칠 수 있습니다.

### 전체 브라켓 전에 만든 시험편

전체 출력 전에 아래 요소만 포함한 작은 시험편을 먼저 만들었습니다.

- 목표 지름을 `0.1~0.2 mm` 간격으로 바꾼 나사 구멍 3~5개
- 실제 나사와 너트가 들어가는 보스 한 개
- PCB 두께에 맞춘 짧은 레일 조각
- 필요하면 FFC 또는 USB 플러그가 통과하는 홈 한 개

시험편에서는 나사가 손으로 들어가는지, 너무 헐겁지 않은지, 너트와 드라이버가 뒤쪽에서 들어가는지, PCB가 레일 홈 바닥에 닿는지, 케이블이 실제 방향으로 빠져나오는지를 확인했습니다. 구멍 하나만 조금 틀어져도 큰 브라켓 전체를 다시 출력하게 되므로 이 단계가 가장 빠른 검증 방법입니다.

## Sindoh 3DWOX 2X 출력 설정

### 프린터 제약

이번 출력은 Sindoh 3DWOX 2X와 PLA를 기준으로 잡았습니다.

| 항목 | 기준 |
| --- | --- |
| 노즐 | `0.4 mm` |
| 필라멘트 | `1.75 mm` |
| 최대 출력 크기, 노즐 1 | `228 × 200 × 300 mm` |
| 최대 출력 크기, 노즐 2 | `225 × 200 × 300 mm` |
| 레이어 높이 범위 | `0.05~0.40 mm` |

긴 베이스는 평평하게 두면 베드 폭을 넘을 수 있습니다. 이 경우 가장 긴 축을 Z 방향으로 세우는 배치를 검토하고, Z 높이 `300 mm` 안에 들어가는지와 서포트·raft 필요성을 같이 확인합니다.

### PLA 시작 프로파일

| 설정 | 시작값 |
| --- | --- |
| 노즐 / 베드 온도 | `200 C / 60 C` |
| 레이어 / 초기 레이어 높이 | `0.20 mm / 0.20 mm` |
| 벽 / 상하단 두께 | `0.80 mm 이상 / 0.80 mm 이상` |
| 인필 | `15%`, 첫 실사용 부품은 `20%`부터 확인 |
| 일반 / 초기 레이어 속도 | `40 mm/s / 20 mm/s` |
| 이동 속도 | `130 mm/s` |
| 리트랙션 | `6.00 mm`, `30 mm/s` |
| 냉각 팬 | `50%` |
| 최소 레이어 시간 | `5초` |

### 서포트와 raft 시작 프로파일

| 설정 | 시작값 |
| --- | --- |
| 서포트 | 켬 |
| 배치 | 우선 `Touching buildplate`, 내부 지붕이 뜨면 `Everywhere` |
| 패턴 / overhang 각도 | `Zigzag / 60도` |
| 밀도 | `20%` |
| XY / Z 간격 | `0.80 mm / 0.20 mm` |
| 베드 부착 | `Raft` |
| Raft 여유 | `5.00 mm` |
| Raft base / interface / surface 속도 | `20 mm/s` |

`Everywhere` 서포트는 내부 지붕을 받쳐 주지만 FFC 창, 나사 구멍, 너트 공간에 남을 수 있습니다. 서포트를 켠 뒤에는 반드시 레이어 미리보기에서 기능 구멍이 막히지 않는지 확인했습니다.

## 출력 및 조립 순서

1. 3DWOX Desktop에서 프린터를 `3DWOX 2X`, 재료를 PLA로 선택합니다.
2. STL을 배치하고 베드 XY 크기와 Z 높이를 먼저 확인합니다.
3. 출력 방향, 서포트, raft를 적용합니다.
4. 슬라이싱 후 레이어 미리보기에서 레일 홈, 고깔 보스 관통 구멍, FPGA 체결 구멍, FFC 창, 너트 공간을 확인합니다.
5. 같은 Wi-Fi에 연결된 프린터는 3DWOX Desktop 또는 앱에서 탐지해 전송할 수 있습니다. USB로 옮길 때도 이 프린터·이 PLA 프로파일로 만든 G-code만 사용합니다.
6. 프린터를 1~2주 사용하지 않았다면 필라멘트 로딩, 노즐 토출, 남은 재료, 베드 청결을 확인합니다. 토출이 불안하면 purge 또는 작은 시험 출력부터 합니다.
7. 전체 브라켓 전에 나사 구멍, 보스, 짧은 레일이 들어간 시험편을 출력하고 실제 PCB·나사·너트로 체결합니다.
8. 시험편이 맞으면 베이스 1개와 카메라 홀더 2개를 출력한 뒤, 렌즈 중심 높이와 케이블 출구를 다시 확인합니다.

## 조립 전 점검표

### CAD 렌더에서 볼 것

- [ ] PCB 나사 구멍 중심과 브라켓 보스 중심이 일치하는가
- [ ] 카메라 PCB 바닥이 **하단 레일 홈 바닥면**에 닿는가
- [ ] 나사가 보스를 완전히 관통하고, 뒤쪽에 너트·와셔·드라이버 공간이 있는가
- [ ] 레일과 리브가 커넥터, 납땜부, 렌즈 하우징을 누르지 않는가
- [ ] FFC·USB·전원 케이블의 삽입과 굽힘 방향이 열려 있는가
- [ ] 두 카메라의 렌즈 중심이 같은 높이와 방향에 있는가
- [ ] baseline을 렌즈 중심 기준으로 계산했는가

### 슬라이서 레이어 미리보기에서 볼 것

- [ ] 레일, 보스, 지붕 아래에 필요한 서포트가 생성되는가
- [ ] 나사 구멍, 너트 공간, FFC 창이 모델 재료나 서포트로 막히지 않는가
- [ ] 모델이 베드 크기와 Z 높이 제한 안에 들어가는가
- [ ] raft가 필요한 면에 충분한 여유를 두고 생성되는가

## 마무리

PCB 브라켓은 치수만 맞춘 상자가 아니라 실제 조립 동작을 담는 부품입니다. 보드가 어느 면에 닿는지, 나사를 어느 쪽에서 넣는지, 너트와 드라이버가 어디에 들어가는지, 케이블이 어느 방향으로 빠져야 하는지를 더미 모델 단계에서 먼저 잡았습니다.

이번 V4 세트에서는 카메라 홀더와 FPGA 베이스를 분리하고, 하단 레일 홈과 고깔형 보스를 먼저 시험 출력한 방식이 가장 효과적이었습니다. 다음에 다른 보드나 센서를 고정할 때도 이 순서대로 확인하면 전체 출력 실패와 재료 낭비를 줄일 수 있습니다.
